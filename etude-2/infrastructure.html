데이터 흐름 — 단계별 설명

1

terraform apply — 인프라 한 번에 프로비저닝

Secrets Manager 시크릿, ECR 리포지토리, EC2 인스턴스 + IAM Role, 빌드서버용 IAM User, GitHub Actions Secrets 8개가 한 번의 apply로 생성됩니다. etude-1의 시크릿 구성이 그대로 이어집니다.

2

EC2 user_data — 부팅 시 초기 설정

인스턴스 최초 기동 시 user_data 스크립트가 실행됩니다. Docker를 설치하고, ECR에 이미 이미지가 있으면 바로 docker run합니다. 이미지가 없으면(최초 배포 전) skip합니다.

3

release 브랜치 push → 워크플로우 트리거

etude-2/springboot/** 경로가 변경된 커밋이 release 브랜치에 push되면 deploy.yml이 실행됩니다. self-hosted runner인 라즈베리파이가 작업을 수행합니다.

4

buildx 크로스 빌드 + ECR push

라즈베리파이에서 docker buildx build --platform linux/amd64로 EC2용 amd64 이미지를 빌드합니다. :latest와 :<commit-sha> 두 태그로 ECR에 push합니다.

5

SSM SendCommand → EC2 재배포

aws ssm send-command로 EC2에 셸 명령을 원격 실행합니다. EC2는 ECR에서 새 이미지를 pull하고 기존 컨테이너를 교체합니다. SSH 접속 없이 배포가 완료됩니다.

6

Spring Boot → Secrets Manager 조회

컨테이너가 기동되면 GET /secret 요청 시 SecretsManagerClient가 시크릿을 읽어 반환합니다. EC2 IAM Role의 자격증명 체인이 자동으로 사용되므로 access key를 컨테이너 환경변수에 넣지 않아도 됩니다.

IAM 권한 상세

EC2 IAM Role (etude2-ec2-app-role)

  • AmazonEC2ContainerRegistryReadOnly — ECR에서 이미지 pull
  • AmazonSSMManagedInstanceCore — SSM Session Manager 접속 허용
  • 인라인 정책 — secretsmanager:GetSecretValue + DescribeSecret (해당 시크릿 ARN만)

Builder IAM User (etude2-builder)

  • ecr:GetAuthorizationToken — ECR 로그인 토큰 발급 (전체 리소스)
  • ecr:BatchCheck / PutImage / Upload… — 해당 ECR 레포에만 push/pull
  • ssm:SendCommand / GetCommandInvocation / ListCommandInvocations — EC2 원격 배포

Spring Boot 핵심 코드

etude-1과 달리 리전과 시크릿 이름을 환경변수(AWS_REGION, AWS_SECRET_NAME)로 주입받으므로 코드 수정 없이 재배포 가능합니다.

@RestController
public class SecretsController {

    private final SecretsManagerClient client;
    private final String secretId;

    public SecretsController(
            @Value("${app.aws.region:ap-northeast-2}") String region,
            @Value("${app.aws.secret-name:etude-2/app-password}") String secretId) {
        this.client = SecretsManagerClient.builder()
                .region(Region.of(region))
                .build();
        this.secretId = secretId;
    }

    @GetMapping("/")      public String hello()     { return "etude-2 ok"; }
    @GetMapping("/health") public String health()    { return "ok"; }

    @GetMapping("/secret")
    public String getSecret() {
        return client.getSecretValue(
            GetSecretValueRequest.builder().secretId(secretId).build()
        ).secretString();  // EC2 IAM Role 자격증명 체인 사용 (access key 불필요)
    }
}

사전 준비 및 실행

1. terraform.tfvars 작성

github_owner      = "your-github-username"
github_repository = "your-repo-name"
# ec2_key_name = "my-key"  # SSH 접속 필요 시

2. Terraform 적용

export GITHUB_TOKEN=ghp_xxx
cd etude-2/terraform
terraform init && terraform apply

3. 라즈베리파이 self-hosted runner 설정 (인프라 외)

# 1) Tailscale 설치 및 ACL 설정
# 2) Docker + buildx 설치
# 3) GitHub repo → Settings → Actions → Runners → 토큰 발급
./config.sh --url https://github.com/<owner>/<repo> --token <TOKEN> \
  --labels "self-hosted,linux,raspberry"
./svc.sh install && ./svc.sh start

4. 동작 확인

# 앱 기동 확인
curl "$(terraform -chdir=terraform output -raw app_url)/health"
# → ok

# Secrets Manager 조회 확인 (EC2 IAM Role 경유)
curl "$(terraform -chdir=terraform output -raw app_url)/secret"
# → {"password":"Ax3!..."}

# key pair 없이 EC2 접속 (SSM Session Manager)
aws ssm start-session --target <EC2_INSTANCE_ID>
⚠️ 정리: terraform destroy 실행 시 EC2, ECR, IAM User(access key 포함), Secrets Manager 시크릿이 모두 삭제됩니다. ECR force_delete = true이므로 이미지가 남아 있어도 강제 삭제됩니다.
