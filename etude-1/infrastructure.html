etude-1 인프라 가이드

Terraform으로 시크릿을 프로비저닝하고 Spring Boot에서 조회하는 전체 흐름

전체 아키텍처

데이터 흐름 — 단계별 설명

1

비밀번호 생성 (random_password)

Terraform이 random 프로바이더로 길이 24, 특수문자 포함 랜덤 비밀번호를 메모리 안에서 생성합니다. 이 값은 state에만 존재하며 외부로 직접 노출되지 않습니다.

2

AWS Secrets Manager에 저장

aws_secretsmanager_secret으로 시크릿 항목을 만들고, aws_secretsmanager_secret_version으로 {"password":"..."} 형식의 JSON 값을 주입합니다. recovery_window_in_days = 0이므로 terraform destroy 시 즉시 삭제됩니다.

3

GitHub Actions 시크릿 주입

github 프로바이더가 동일 apply 내에서 3개의 Actions 시크릿을 대상 레포지토리에 씁니다 — AWS_SECRET_NAME, AWS_REGION(어디서 읽을지), APP_PASSWORD(평문 비밀번호). CI가 AWS 자격증명 없이도 비밀번호를 직접 참조할 수 있도록 하기 위함입니다.

4

Spring Boot에서 조회

SecretsController가 AWS SDK SecretsManagerClient를 통해 GET / 요청 시 시크릿을 조회하고 JSON 문자열을 응답으로 반환합니다. 자격증명은 DefaultCredentialsProviderChain에 의해 자동 선택됩니다.

5

자격증명 자동 선택

SDK가 환경에 따라 자격증명 소스를 자동으로 결정합니다. 로컬 실행 시에는 ~/.aws/credentials의 terraform_user 프로필이 사용되고, GitHub Actions에서는 OIDC 또는 주입된 Access Key가 사용됩니다.

사전 준비 및 실행

1. AWS IAM 설정

로컬 aws-cli에 SecretsManagerReadWrite 권한을 가진 IAM 사용자 terraform_user 프로필이 필요합니다.

2. GitHub PAT 노출

export GITHUB_TOKEN=ghp_xxx   # repo 스코프 필요

3. terraform.tfvars 작성

github_owner      = "your-github-username"
github_repository = "your-repo-name"

4. Terraform 적용

cd etude-1/terraform
terraform init
terraform plan
terraform apply

5. Spring Boot 실행 및 확인

cd etude-1/springboot
./gradlew bootRun

# 새 터미널에서
curl http://localhost:8080/
# → {"password":"Ax3!..."}
⚠️ 정리: terraform destroy 실행 시 recovery_window_in_days = 0 설정으로 인해 시크릿이 즉시 영구 삭제됩니다.

Spring Boot 핵심 코드

@RestController
public class SecretsController {

    private final SecretsManagerClient client =
        SecretsManagerClient.builder()
            .region(Region.AP_NORTHEAST_2)
            .build();  // DefaultCredentialsProviderChain 사용

    @GetMapping("/")
    public String getSecrets() {
        GetSecretValueResponse response = client.getSecretValue(
            GetSecretValueRequest.builder()
                .secretId("etude-1/app-password")
                .build()
        );
        return response.secretString();  // {"password":"..."}
    }
}
